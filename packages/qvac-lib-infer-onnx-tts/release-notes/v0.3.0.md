# QVAC TTS ONNX Addon v0.3.0 Release Notes

This major release introduces the Chatterbox TTS engine to the QVAC TTS ONNX addon, expanding the library's capabilities beyond Piper to support voice cloning and multilingual synthesis. The addon now automatically detects and configures the appropriate engine based on the provided configuration, making it simple to work with either Piper or Chatterbox models.

## New Features

### Chatterbox TTS Engine with Voice Cloning

The addon now supports Chatterbox, an advanced neural TTS engine that enables voice cloning from reference audio. Chatterbox uses a transformer-based language model architecture with five ONNX models working together to produce natural-sounding speech that mimics the characteristics of a reference voice sample.

To use Chatterbox, provide the following configuration when initializing `ONNXTTS`:

```js
const ONNXTTS = require('@qvac/tts-onnx')
const { readWavAsFloat32 } = require('./wav-generator-helper')

// Load reference audio for voice cloning (24kHz, Float32Array)
const { samples } = readWavAsFloat32('path/to/reference.wav')

const model = new ONNXTTS({
  tokenizerPath: 'models/chatterbox/tokenizer.json',
  speechEncoderPath: 'models/chatterbox/speech_encoder.onnx',
  embedTokensPath: 'models/chatterbox/embed_tokens.onnx',
  conditionalDecoderPath: 'models/chatterbox/conditional_decoder.onnx',
  languageModelPath: 'models/chatterbox/language_model.onnx',
  referenceAudio: samples, // Float32Array of reference audio samples
  opts: { stats: true }
}, {
  language: 'en' // Supports: en, es, fr, de, it, pt, ru
})

await model.load()
const response = await model.run({
  input: 'Hello world! This is voice cloning.',
  type: 'text'
})
```

Chatterbox supports seven languages: English, Spanish, French, German, Italian, Portuguese, and Russian. The synthesized audio is output at 24kHz sample rate.

### Automatic Engine Detection

The library now automatically selects the appropriate TTS engine based on the configuration parameters you provide. If you specify `tokenizerPath`, `speechEncoderPath`, or other Chatterbox-specific fields, the Chatterbox engine is used. Otherwise, the addon defaults to the Piper engine. This removes any need for manual engine selection.

### TypeScript Enhancements

The TypeScript definitions have been expanded to support both engines with clear, documented interfaces:

- `PiperTTSArgs`: Configuration for Piper engine (existing)
- `ChatterboxTTSArgs`: Configuration for Chatterbox engine (new)
- `ONNXTTSArgs`: Union type that accepts either configuration

All interfaces now include JSDoc comments explaining each field, making it easier to understand the required configuration options through IDE autocomplete.

### Comprehensive Test Coverage

Integration tests for Chatterbox have been added to validate:
- Basic text synthesis with voice cloning
- Multiple sentence processing
- Reference audio handling and validation
- Model loading and unloading lifecycle
- Performance metrics and audio quality checks

Unit tests cover the C++ implementation of the Chatterbox inference pipeline, including tokenization, speech encoding, conditional decoding, and language model generation.

## Build and Infrastructure Improvements

### Rust Target Support for Mobile Platforms

The CI/CD pipeline now automatically installs the necessary Rust targets for cross-platform builds:
- iOS arm64 device and simulator targets
- Android arm64 and armv7 targets

This ensures the `tokenizers-cpp` dependency compiles successfully on all mobile platforms without manual intervention.

### Updated Dependencies

The following dependencies have been updated to support Chatterbox:
- Added `tokenizers-cpp` (v0.1.1) for BPE tokenization used by Chatterbox
- Updated vcpkg baseline to include the latest ports
- Updated `qvac-lib-inference-addon-cpp` to v0.12.2

### Windows Build Fixes

Several Windows-specific build issues have been resolved, improving cross-platform reliability of the build system.

## Bug Fixes

### Reference Audio Handling in Integration Tests

The Piper integration tests previously referenced a `ref.wav` file that wasn't used by the Piper engine, causing confusion. This has been cleaned up. The Chatterbox tests now properly use reference audio for voice cloning, while Piper tests no longer reference unused audio files.

## API and Implementation Details

### New C++ Infrastructure

Several new C++ components power the Chatterbox engine:

- `OnnxInferSession`: A universal ONNX model inference session wrapper that simplifies running models with dynamic input/output shapes
- `ChatterboxEngine`: Main inference engine implementing the five-stage Chatterbox pipeline (tokenization, speech encoding, embedding, conditional decoding, and language model generation)
- `AudioResult`: Shared data structure for audio output across both Piper and Chatterbox engines

### JavaScript Interface Updates

The main JavaScript interface (`index.js`) now includes:
- Engine detection logic based on configuration fields
- Separate path resolution for Chatterbox and Piper model files
- Support for passing `Float32Array` reference audio to the C++ addon
- Enhanced reload functionality that preserves the selected engine

### Example Applications

A new example application (`examples/example-chatterbox-tts.js`) demonstrates how to:
- Load reference audio from a WAV file
- Initialize the Chatterbox TTS engine with voice cloning
- Run synthesis and save the output
- Handle errors and resource cleanup

## Testing and Quality Assurance

All existing tests continue to pass, and the new Chatterbox tests have been integrated into the CI pipeline. The test suite now covers both Piper and Chatterbox engines with mobile and desktop platforms validated through automated testing.

## Notes

- The `.cache/` directory is now ignored by Git to prevent build artifacts from being committed
- Reference audio for Chatterbox examples (`examples/ref.wav`) is now tracked in Git for demonstration purposes
- Chatterbox synthesis is computationally intensive and may take longer than Piper, especially on mobile devices
